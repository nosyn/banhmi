# Wave 4 ā€” Data Implementation Plan

**Goal:** Ship 4 data packages ā€” postgres, mysql, drizzle, mongo.

**Architecture:** Each package follows the established module/repository pattern. Connection providers via `forRoot`; per-entity repos via `forFeature`. All Bun-native where possible (`Bun.sql` for postgres+mysql, `bun:sqlite` for drizzle's sqlite driver).

**Cross-task conventions:** identical to Waves 1ā€“3. Reference `packages/sqlite/` for the existing pattern.

**Order of work:**

1. `@banhmi/postgres` ā€” uses `Bun.sql`.
2. `@banhmi/mysql` ā€” same pattern with mysql adapter.
3. `@banhmi/drizzle` ā€” wraps drizzle-orm, drivers for sqlite + postgres-js.
4. `@banhmi/mongo` ā€” wraps `mongodb` driver.
5. Cluster app updates: `examples/drizzle-api/` upgrade + new `examples/cats-mongo/`.
6. Wave 4 verification gate + canary.

## Task 1 ā€” `@banhmi/postgres`

Files:

```
packages/postgres/
  package.json (deps: banhmi, @banhmi/common, @banhmi/core)
  tsconfig.json
  bunfig.toml
  src/index.ts
  src/postgres.module.ts
  src/repository.ts                  # base Repository class + @Repository decorator
  src/inject-postgres.ts             # @InjectPostgres() / @InjectSql()
  src/types.ts
  src/tokens.ts
  test/repository.test.ts
  test/postgres.module.test.ts
  test/integration.test.ts           # gated on DATABASE_URL

examples/features/postgres/{...}
```

API:

```ts
export { PostgresModule } from './postgres.module'
export { Repository, BaseRepository } from './repository'
export { InjectSql } from './inject-postgres'
export type { PostgresOptions, Sql } from './types'
export { POSTGRES_SQL } from './tokens'
```

```ts
type PostgresOptions = {
  url?: string                                  // default Bun.env.DATABASE_URL
  hostname?: string
  database?: string
  user?: string
  password?: string
  max?: number                                  // pool size
}
```

`PostgresModule.forRoot(opts)` returns a DynamicModule providing `POSTGRES_SQL` (= a `Bun.sql` template tag). `forFeature(entities)` registers per-entity repositories.

`@Repository(EntityClass)` (parallel to `packages/sqlite/src/sqlite.module.ts`'s `@Repository`) generates a class with `findById`, `findAll`, `insert`, `update`, `delete`. Use `Bun.sql`'s tagged-template safety.

Tests:
- Module registers `POSTGRES_SQL`.
- Mocked `Sql` for unit tests of `BaseRepository.findById` queries.
- Integration test under `if (Bun.env.DATABASE_URL)` runs CRUD against a real DB.

Micro-example: app with a `User` repo, hitting Postgres if `DATABASE_URL` is set; else asserts module registration only.

Doc page: `apps/docs/apps/web/src/content/techniques/database.mdx` (extend existing if it documents sqlite; cover both sqlite + postgres there).

Re-export.

Commit: `feat(postgres): add @banhmi/postgres package backed by Bun.sql`


Public API:

```ts
export { DrizzleModule } from './drizzle.module'
export { InjectDb } from './inject-db'
export { sqliteDriver, postgresJsDriver } from './drivers'
export type { DrizzleOptions } from './types'
export { DRIZZLE_DB } from './tokens'
```

```ts
type DrizzleOptions = {
  driver: ReturnType<typeof sqliteDriver | typeof postgresJsDriver>
  schema?: unknown                                // user's schema export
}
```

```ts
sqliteDriver({ filename: string }): DrizzleDriver
postgresJsDriver({ url: string, max?: number }): DrizzleDriver
```

`DrizzleModule.forRoot({ driver, schema })` initializes the drizzle instance and provides it as `DRIZZLE_DB`.

Tests:
- sqlite driver creates a working drizzle instance against an in-memory SQLite DB.
- postgres-js driver registers correctly (mocked client).
- `@InjectDb()` returns the drizzle instance from DI.

Micro-example: drizzle schema + a service that queries via `db.select()`. Test runs against in-memory SQLite via the `sqliteDriver`.

Doc page: a new entry under recipes ā€” `apps/docs/apps/web/src/content/recipes/drizzle.mdx`. Update `doc-routes.json` to add it under `recipes`.

Public API:

```ts
export { MongoModule } from './mongo.module'
export { Repository, MongoRepository } from './repository'
export { InjectMongo, InjectCollection } from './inject-mongo'
export type { MongoOptions } from './types'
export { MONGO_DB } from './tokens'
```

```ts
type MongoOptions = {
  url: string                  // mongodb://...
  database: string
}
```

`MongoModule.forRoot(opts)` connects via `MongoClient` from `mongodb` peer; provides `MONGO_DB` token.
`forFeature(entities)` registers `@Repository(Entity)` providers ā€” each is an `MongoRepository<T>` wrapping a `Collection<T>` with `findById`, `findOne`, `find`, `insertOne`, `updateOne`, `deleteOne`.

Tests:
- Module registers Db (mocked `MongoClient`).
- Repository CRUD against an in-memory Mongo (use `mongodb-memory-server` or skip integration if unavailable; default to mocked tests).
- Integration test gated on `MONGO_URL`.

Micro-example: app with a `Cat` repo. Test asserts module registration; integration runs CRUD if `MONGO_URL` set.

Doc page: `apps/docs/apps/web/src/content/techniques/mongo.mdx` (already exists in IA).

Commit: `feat(mongo): add @banhmi/mongo with native driver wrapper`
