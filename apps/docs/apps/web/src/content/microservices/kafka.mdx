# Kafka Transport

**Tier C — stub only.** A full implementation is deferred to wave 9.

## Status

`kafkaTransport()` exports a `KafkaTransport` class that implements the
`Transport` interface but **throws at runtime** with a clear error message.

```
[KafkaTransport] Not implemented (Tier C stub).
kafkajs has Bun compatibility issues (missing Node.js native modules for
TLS/admin). Use customTransport() instead.
TODO(wave-9): implement when kafkajs ships a Bun-native build.
```

## Why deferred

`kafkajs` depends on Node.js `tls`, `net`, and admin-API modules that are not
fully polyfilled in Bun 1.3.x. The package can be installed but throws at
runtime when establishing a broker connection.

## Workaround

Use `customTransport()` with a Kafka client that is Bun-compatible:

```ts
import { customTransport } from '@banhmi/microservices'

const transport = customTransport({
  async listen(handler) { /* use your Kafka client here */ },
  async close() { /* cleanup */ },
  async send(pattern, data) { return { data: null } },
  async emit(pattern, data) {},
})
```

## API reference (stub)

- `kafkaTransport(opts?)` — creates a `KafkaTransport` stub.
- `KafkaTransportOptions.brokers` — Kafka broker addresses.
- `KafkaTransportOptions.groupId` — consumer group id.
- `KafkaTransportOptions.prefix` — topic prefix.
