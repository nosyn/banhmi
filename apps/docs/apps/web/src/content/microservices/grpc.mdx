# gRPC Transport

**Tier C — stub only.** A full implementation is deferred to wave 9.

## Status

`grpcTransport()` exports a `GrpcTransport` class that implements the
`Transport` interface but **throws at runtime** with a clear error message.

```
[GrpcTransport] Not implemented (Tier C stub).
@grpc/grpc-js requires native addons not yet stable in Bun.
Use customTransport() instead.
TODO(wave-9): implement when @grpc/grpc-js is Bun-compatible.
```

## Why deferred

`@grpc/grpc-js` requires native Node.js addons (`grpc_node.node`) for TLS and
HTTP/2 framing. Bun's native-addon loading is not yet stable enough to support
these in production.

## Workaround

Use `customTransport()` with a gRPC library that works under Bun's environment,
or use the TCP transport and define your own serialisation layer:

```ts
import { customTransport } from '@banhmi/microservices'

const transport = customTransport({
  async listen(handler) { /* gRPC-compatible impl */ },
  async close() { /* cleanup */ },
  async send(pattern, data) { return { data: null } },
  async emit(pattern, data) {},
})
```

## API reference (stub)

- `grpcTransport(opts?)` — creates a `GrpcTransport` stub.
- `GrpcTransportOptions.url` — gRPC server address.
- `GrpcTransportOptions.protoPath` — path to the `.proto` file.
- `GrpcTransportOptions.package` — fully-qualified gRPC package name.
