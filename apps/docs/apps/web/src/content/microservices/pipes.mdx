# Pipes

Pipes transform or validate the payload before it reaches the handler.
In microservice contexts, the payload is the first argument passed to the handler.

## When to use

- You want to validate or coerce incoming message payloads.
- You need to transform raw data (e.g. parse a date string) before the handler runs.

## Note on microservice pipes

`@banhmi/microservices` does not inject pipes automatically into the handler
call path — payload transformation happens in the handler itself (first arg).

To compose validation/transformation, call your pipe's `transform()` inside the
handler, or use `runMsEnhancerPipeline()` with interceptors to pre-process the
payload before it reaches the handler.

## Example — validation inside handler

```ts
import { Injectable } from '@banhmi/common'
import { MessagePattern } from '@banhmi/microservices'
import { ValidationPipe } from '@banhmi/validation'

const pipe = new ValidationPipe()

@Injectable()
class CatsHandler {
  @MessagePattern('cats.create')
  async create(payload: unknown) {
    const dto = await pipe.transform(payload, { type: 'body' })
    return this.cats.create(dto)
  }
}
```

## API reference

- `PipeTransform<T, R>` — `transform(value: T, metadata: PipeMetadata): R | Promise<R>`.
- `PipeMetadata` — `{ type, data? }`.
