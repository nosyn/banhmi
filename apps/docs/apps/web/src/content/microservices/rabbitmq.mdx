# RabbitMQ Transport

**Tier B** — ships with `@banhmi/microservices`. Requires `amqplib` as a peer dependency.

## When to use

- You need reliable message delivery with acknowledgements.
- You already operate a RabbitMQ broker.
- You want persistent queues and exchange routing.

## Setup

```bash
bun add amqplib
```

```ts
import { MicroserviceModule, rabbitMqTransport } from '@banhmi/microservices'
import { Module } from '@banhmi/common'

@Module({
  imports: [
    MicroserviceModule.forRoot({
      transport: rabbitMqTransport({ url: 'amqp://localhost', queue: 'cats' }),
    }),
  ],
  providers: [CatsHandler],
})
class AppModule {}
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `url` | `string` | `'amqp://localhost'` | AMQP broker URL |
| `queue` | `string` | `'banhmi_ms'` | Queue name to consume from / publish to |

## Usage

```ts
const client = new ClientProxy(
  rabbitMqTransport({ url: 'amqp://localhost', queue: 'cats' })
)
const cat = await client.send<Cat>('cats.findOne', '1')
await client.close()
```

## Reply queues

`send()` creates a per-request exclusive auto-delete reply queue.
The queue name is passed as `replyTo` in the AMQP message properties,
correlated via `correlationId`.

## API reference

- `rabbitMqTransport(opts?)` — creates a `RabbitMqTransport`.
- `RabbitMqTransportOptions.url` — AMQP broker URL.
- `RabbitMqTransportOptions.queue` — target queue name.
