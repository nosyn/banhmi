# Guards

Microservice guards run before the handler and can reject a message.
When a guard returns `false` (or a rejected promise), `ForbiddenException` is thrown
and the pipeline returns `{ error: { message: 'Forbidden', status: 403 } }`.

## When to use

- You need authentication/authorisation on inbound microservice messages.
- You want to gate access to patterns based on metadata in the message.

## Defining a guard

```ts
import type { MsGuard, MsExecutionContext } from '@banhmi/microservices'

class AuthGuard implements MsGuard {
  canActivate(ctx: MsExecutionContext): boolean {
    const msg = ctx.getMessage()
    // Check for an auth token in the message data
    const data = msg.data as { token?: string }
    return data.token === 'secret'
  }
}
```

## Using the enhancer pipeline

```ts
import { runMsEnhancerPipeline } from '@banhmi/microservices'

const result = await runMsEnhancerPipeline(
  ctx,
  () => handler(msg),
  [new AuthGuard()],  // guards
  [],                  // interceptors
  [],                  // filters
)
```

## API reference

- `MsGuard` — `canActivate(ctx: MsExecutionContext): boolean | Promise<boolean>`.
- `MsExecutionContext` — `getMessage()`, `getClass()`, `getHandler()`.
- `runMsEnhancerPipeline()` — runs guards → interceptors → handler → filters.
