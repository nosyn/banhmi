# Guards

Guards implement the `CanActivate` interface and decide whether a request should
proceed to the handler. They run **after** middleware and **before** interceptors.

## When to use

- Enforce authentication (JWT verification, session lookup).
- Enforce authorisation (role checks, permission gates).
- Block requests that violate business invariants before they reach handlers.

## Setup

```ts
// Global
app.useGlobalGuards(new AuthGuard())

// Controller-level
@Controller('cats')
@UseGuards(AuthGuard)
export class CatsController {}

// Handler-level
@Get(':id')
@UseGuards(RolesGuard)
findOne() {}
```

## Usage — custom guard

```ts
import type { CanActivate, ExecutionContext } from 'banhmi'

export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest<Request>()
    const token = request.headers.get('authorization')?.split(' ')[1]
    return verifyJwt(token)
  }
}
```

## Role-based guard example

```ts
import { Reflector } from 'banhmi'
import type { CanActivate, ExecutionContext } from 'banhmi'

@Injectable()
export class RolesGuard implements CanActivate {
  static inject = [Reflector] as const
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const required = this.reflector.get<string[]>('roles', context.getHandler())
    if (!required) return true
    const user = context.switchToHttp().getRequest<Request & { user: User }>().user
    return required.some((r) => user.roles.includes(r))
  }
}
```

## API reference

- `CanActivate` — interface: `canActivate(context): boolean | Promise<boolean>`.
- `@UseGuards(...guards)` — apply guards to a controller or handler.
- `app.useGlobalGuards(...guards)` — apply guards application-wide.
- `ExecutionContext` — wraps the request for HTTP, WS, and microservice contexts.
- `Reflector` — reads metadata set by `@SetMetadata`.
- `@SetMetadata(key, value)` — attach custom metadata for guards to read.
