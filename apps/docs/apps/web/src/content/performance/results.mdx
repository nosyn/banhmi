## Cold-start (ms) — lower is better

Time from `bun run start` to first `200 OK` on `GET /`.

| Competitor | Startup (ms) | Polls |
|---|---|---|
| **elysia** | **2** | 1 |
| **hono** | **104** | 2 |
| **banhmi** | **154** | 2 |
| nestjs-fastify | 307 | 4 |
| nestjs-express | 309 | 4 |

**Banhmi is 2× faster to start than NestJS** (154 ms vs 307–309 ms).
Elysia and Hono have near-zero startup because they avoid module initialisation overhead.

---

## RSS idle (kB) — lower is better

Resident Set Size after 10 seconds of idle (no traffic).

| Competitor | RSS (kB) | RSS (MB) |
|---|---|---|
| **hono** | **42 992** | **42** |
| **banhmi** | **63 488** | **62** |
| elysia | 78 272 | 76 |
| nestjs-express | 83 424 | 81 |
| nestjs-fastify | 83 584 | 82 |

**Banhmi uses 24% less memory than NestJS at idle** (62 MB vs 81–82 MB).

## Raw JSON results

All result files live under [`benchmarks/results/2026-05-09/`](https://github.com/banhmi/banhmi/tree/main/benchmarks/results/2026-05-09).
Each file is named `<scenario>-<competitor>.json` and contains the structured output
from the runner (rps, p50, p95, p99, p99_9, totalSeconds) or a `note` field when
the runner was unavailable.
