# Exception Filters

WebSocket exception filters catch errors thrown inside `@SubscribeMessage`
handlers and send a structured error event back to the client instead of
crashing the connection.

## When to use

- Convert domain exceptions into a consistent WebSocket error-event shape.
- Prevent unhandled promise rejections from terminating the WebSocket connection.

## Usage

```ts
import { Catch, ArgumentsHost, ExceptionFilter } from 'banhmi'
import type { WsException } from 'banhmi'

@Catch(WsException)
export class WsExceptionFilter implements ExceptionFilter {
  catch(exception: WsException, host: ArgumentsHost) {
    const client = host.switchToWs().getClient()
    client.send(JSON.stringify({
      event: 'error',
      data: { message: exception.message },
    }))
  }
}
```

Apply at the gateway level:

```ts
@UseFilters(WsExceptionFilter)
@WebSocketGateway({ path: '/ws' })
export class ChatGateway {}
```

## API reference

- `WsException(message | object)` — the base WebSocket exception class.
- `@Catch(...types)` — bind the filter to specific exception classes.
- `host.switchToWs().getClient()` — the raw `BunWebSocket` to send back to.
- `host.switchToWs().getData()` — the payload that triggered the handler.
- `@UseFilters(...filters)` — apply filters to a gateway or message handler.
