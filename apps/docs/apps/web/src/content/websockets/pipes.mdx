# Pipes

Pipes in WebSocket gateways work exactly like HTTP pipes. They transform or
validate the `@MessageBody()` before the handler is invoked.

## When to use

- Validate the shape of incoming WebSocket payloads.
- Parse numeric or UUID fields from message data.

## Usage

```ts
import { ZodValidationPipe } from '@banhmi/validation'
import { z } from 'zod'

const SendMessageSchema = z.object({
  room: z.string().min(1),
  text: z.string().min(1).max(500),
})

@SubscribeMessage('message')
handleMessage(
  @MessageBody(new ZodValidationPipe(SendMessageSchema))
  payload: z.infer<typeof SendMessageSchema>,
) {
  this.server.to(payload.room).emit('message', payload.text)
}
```

Apply globally to all gateway message handlers:

```ts
@WebSocketGateway({ path: '/ws' })
@UsePipes(new ZodValidationPipe())
export class ChatGateway {}
```

## API reference

- `@MessageBody(pipe?)` — extract and optionally pipe the message body.
- `@UsePipes(...pipes)` — apply pipes to all handlers in the gateway.
- `ZodValidationPipe(schema?)` — Zod-based validation; throws `WsException` on failure.
