# Built-in Exceptions

All built-in HTTP exceptions extend the base `HttpException` class. Throw them
from any handler, guard, pipe, or interceptor and Banhmi's default exception
filter converts them to the correct HTTP status.

## Usage

```ts
import { NotFoundException } from 'banhmi'

@Get(':id')
findOne(@Param('id') id: string) {
  const cat = this.catsService.findOne(id)
  if (!cat) throw new NotFoundException(`Cat #${id} not found`)
  return cat
}
```

## Custom exception

```ts
import { HttpException, HttpStatus } from 'banhmi'

export class DomainException extends HttpException {
  constructor(message: string, code: string) {
    super({ message, code }, HttpStatus.UNPROCESSABLE_ENTITY)
  }
}
```

## API reference

- `HttpException(response, status)` — base class; `response` can be a string or object.
- `exception.getStatus()` — returns the HTTP status code.
- `exception.getResponse()` — returns the response body (string or object).
