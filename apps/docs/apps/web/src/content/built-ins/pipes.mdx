# Built-in Pipes

Banhmi ships a set of built-in pipes that cover the most common transformation
and validation scenarios. All built-in pipes are exported from `banhmi`.

## ParseIntPipe

Converts a string param to a JavaScript integer. Throws `BadRequestException`
if the value is not a valid integer.

```ts
@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {
  return this.catsService.findOne(id)
}
```

## ParseFloatPipe

Same as `ParseIntPipe` but allows decimal values.

## ParseBoolPipe

Accepts `'true'` / `'false'` and coerces to `boolean`.

## ParseUUIDPipe

Validates that the value is a canonical UUID (v1–v5). Throws on invalid format.

```ts
@Get(':id')
findOne(@Param('id', ParseUUIDPipe) id: string) {}
```

## DefaultValuePipe

Returns a fallback value when the incoming value is `undefined` or `null`.

```ts
@Get()
findAll(
  @Query('page', new DefaultValuePipe(1), ParseIntPipe) page: number,
) {}
```

## ValidationPipe

Validates the body or query object against a Zod schema (or a class with
class-validator decorators when configured). Throws `UnprocessableEntityException`
on failure.

```ts
import { z } from 'zod'

const CreateCatSchema = z.object({
  name: z.string().min(1),
  age: z.number().int().min(0),
})

@Post()
create(@Body(new ValidationPipe(CreateCatSchema)) dto: CreateCatDto) {}
```

## API reference

| Pipe | Input | Output | Error |
|------|-------|--------|-------|
| `ParseIntPipe` | `string` | `number` (integer) | `BadRequestException` |
| `ParseFloatPipe` | `string` | `number` | `BadRequestException` |
| `ParseBoolPipe` | `'true'|'false'` | `boolean` | `BadRequestException` |
| `ParseUUIDPipe` | `string` | `string` (UUID) | `BadRequestException` |
| `DefaultValuePipe(d)` | `T | undefined` | `T` | — |
| `ValidationPipe(schema?)` | `unknown` | validated `T` | `UnprocessableEntityException` |
