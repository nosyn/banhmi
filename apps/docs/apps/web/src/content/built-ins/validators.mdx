# Built-in Validators

The `@banhmi/validation` package provides `ZodValidationPipe` and related
helpers for schema-based input validation.

## ZodValidationPipe

Validates body, query, or param against a Zod schema:

```ts
import { ZodValidationPipe } from '@banhmi/validation'
import { z } from 'zod'

const CreateCatSchema = z.object({
  name: z.string().min(1),
  age: z.number().int().min(0),
})

@Post()
create(
  @Body(new ZodValidationPipe(CreateCatSchema))
  dto: z.infer<typeof CreateCatSchema>,
) {
  return this.catsService.create(dto)
}
```

## Global setup

```ts
app.useGlobalPipes(new ZodValidationPipe())
```

When used globally without a schema argument, the pipe looks for a schema
attached to the body DTO via `@ZodSchema(schema)`.

## @ZodSchema decorator

```ts
import { ZodSchema } from '@banhmi/validation'
import { z } from 'zod'

const Schema = z.object({ name: z.string(), age: z.number() })

@ZodSchema(Schema)
export class CreateCatDto {}
```

## Error format

A validation failure returns HTTP 422 with:

```json
{
  "statusCode": 422,
  "message": "Validation failed",
  "errors": [
    { "path": ["age"], "message": "Expected number, received string" }
  ]
}
```

## API reference

- `ZodValidationPipe(schema?)` — the main validation pipe.
- `@ZodSchema(schema)` — class decorator to attach a schema to a DTO.
- `formatZodError(error)` — utility: converts `ZodError` to the standard error shape.
