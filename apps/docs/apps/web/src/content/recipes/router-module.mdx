# Router Module

The `RouterModule` lets you define path prefixes at the module level,
so controllers inside the module get those prefixes automatically — without
repeating them in every `@Controller('prefix')`.

## Usage

```ts
import { Module } from 'banhmi'
import { RouterModule } from '@banhmi/core'
import { CatsModule } from './cats/cats.module'
import { DogsModule } from './dogs/dogs.module'

@Module({
  imports: [
    CatsModule,
    DogsModule,
    RouterModule.register([
      {
        path: 'v1',
        children: [CatsModule, DogsModule],
      },
    ]),
  ],
})
export class AppModule {}
```

All `CatsController` routes now live under `/v1/cats`.

## Nested routing

```ts
RouterModule.register([
  {
    path: 'admin',
    children: [
      {
        path: 'users',
        module: AdminUsersModule,
      },
    ],
  },
])
```

Routes become `/admin/users/...`.

## API reference

- `RouterModule.register(routes)` — register path prefixes for child modules.
- `RouteTree` — `{ path: string, module?: Type, children?: RouteTree[] }`.
