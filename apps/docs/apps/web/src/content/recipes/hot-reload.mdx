# Hot reload

Bun has built-in watch mode (`bun --watch`) that restarts the process on file
changes. No separate HMR plugin is needed. This is faster and simpler than
Webpack HMR used in Node.js-based setups.

## Setup

Add a `dev` script to `package.json`:

```json
{
  "scripts": {
    "dev": "bun --watch run src/main.ts"
  }
}
```

Or use the Banhmi CLI:

```bash
banhmi start --watch
```

## Behaviour

`bun --watch` performs a **full process restart** on any `.ts` change under `src/`.
The restart is typically under 200ms (including module graph re-resolution).

## Ignoring paths

Use `bunfig.toml` to exclude directories from watch:

```toml
[watch]
ignore = ["dist/**", "*.test.ts"]
```

## What about NestJS HMR?

NestJS uses Webpack HMR (`module.hot`) for partial module replacement. Banhmi
doesn't support this — Bun's cold-start is fast enough (154ms) that full-process
restart is the simpler and more reliable choice.
