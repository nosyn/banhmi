# REPL

The Banhmi REPL (Read-Eval-Print Loop) lets you interact with your
application's DI container from a terminal — call service methods, inspect
provider state, and run ad-hoc queries without writing scripts.

## Setup

Add a `repl.ts` entrypoint:

```ts
// src/repl.ts
import { repl } from 'banhmi'
import { AppModule } from './app.module'

await repl(AppModule)
```

Run it:

```bash
bun run src/repl.ts
```

## Usage in the REPL

```js
// Retrieve a provider
const catsService = await get(CatsService)

// Call a method
await catsService.findAll()

// Inspect available providers
debug()
```

## REPL helpers

| Command | Description |
|---------|-------------|
| `get(Token)` | Retrieve a provider from the DI container |
| `resolve(Token)` | Create a new transient/request-scoped instance |
| `debug()` | List all registered modules and providers |
| `methods(instance)` | List available methods on an instance |
