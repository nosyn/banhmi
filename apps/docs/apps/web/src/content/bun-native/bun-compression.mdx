# Bun.gzip / deflate / zstd

Bun ships built-in gzip, deflate, and zstd compression via `Bun.gzip`,
`Bun.deflate`, `Bun.zstd`, and their decompression counterparts. These are
synchronous Zig implementations with no additional dependencies.

## When to use

- Compress a `Uint8Array` payload before writing it to a cache or S3.
- Decompress a compressed HTTP response body.
- Integrate custom compression into a Banhmi interceptor.

## Usage

```ts
// Compress
const data = new TextEncoder().encode('Hello, world!')
const compressed = Bun.gzipSync(data)

// Decompress
const decompressed = Bun.gunzipSync(compressed)
console.log(new TextDecoder().decode(decompressed)) // Hello, world!
```

## Middleware compression

For automatic HTTP response compression, use `@banhmi/compression`:

```ts
import { compressionMiddleware } from '@banhmi/compression'

const app = await BanhmiFactory.create(AppModule)
app.use(compressionMiddleware())
await app.listen(3000)
```

The middleware negotiates `Accept-Encoding` and applies the best available
encoding (zstd > br > gzip > deflate).

## Async API

```ts
const compressed = await Bun.gzip(data)
const decompressed = await Bun.gunzip(compressed)
```

## API reference

| Function | Description |
|----------|-------------|
| `Bun.gzipSync(data)` | Synchronous gzip compression |
| `Bun.gunzipSync(data)` | Synchronous gzip decompression |
| `Bun.deflateSync(data)` | Synchronous deflate compression |
| `Bun.inflateSync(data)` | Synchronous deflate decompression |
| `Bun.gzip(data)` | Async gzip |
| `Bun.gunzip(data)` | Async gunzip |
