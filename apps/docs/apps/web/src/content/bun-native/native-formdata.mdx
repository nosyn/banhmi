# Native FormData

Bun parses `multipart/form-data` and `application/x-www-form-urlencoded`
natively via the web-standard `Request.formData()` method. Banhmi's
`@banhmi/multipart` wraps this in a provider with type-safe field extraction.

## When to use

- Accept HTML form submissions.
- Receive file uploads from `<input type="file">` elements.

## Native formData (no package)

```ts
@Post('upload')
async upload(@Req() req: Request) {
  const form = await req.formData()
  const file = form.get('file') as File
  const name = form.get('name') as string

  const bytes = await file.arrayBuffer()
  await Bun.write(`./uploads/${file.name}`, bytes)

  return { name, filename: file.name, size: file.size }
}
```

## With @banhmi/multipart

```ts
import { UploadedFile, UseMultipart } from '@banhmi/multipart'

@Post('upload')
@UseMultipart()
async upload(
  @Body('name') name: string,
  @UploadedFile('avatar') avatar: File,
) {
  await Bun.write(`./uploads/${avatar.name}`, await avatar.arrayBuffer())
  return { name, filename: avatar.name }
}
```

## File validation

```ts
@Post('upload')
@UseMultipart({ maxFileSize: 5 * 1024 * 1024, allowedMimeTypes: ['image/*'] })
async upload(@UploadedFile('photo') photo: File) {
  // Only proceeds if mime and size pass
}
```

## API reference

- `Request.formData()` — native Bun API; returns a `FormData` object.
- `FormData.get(name)` — returns a `string` or `File`.
- `FormData.getAll(name)` — returns an array (useful for multi-file inputs).
- `@UseMultipart(options?)` — Banhmi middleware for form parsing.
- `@UploadedFile(field)` — extract a `File` from multipart data.
