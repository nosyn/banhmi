# Bun.serve

`Bun.serve` is the zero-dependency HTTP server built into Bun. Banhmi's
`BunAdapter` wraps it to provide routing, DI, and the full enhancer pipeline,
while keeping raw access available when you need it.

## How Banhmi uses Bun.serve

`BanhmiFactory.create(AppModule)` creates a `BunAdapter` that calls `Bun.serve`
under the hood. The adapter wires the Radix-tree router into the `fetch` handler
and hands off WebSocket upgrades to the `BunWebSocketAdapter`.

You never call `Bun.serve` directly in a Banhmi app — the factory handles it.

## Accessing the underlying server

```ts
const app = await BanhmiFactory.create(AppModule)
const server = await app.listen(3000)
// `server` is the `Bun.Server` instance
console.log(`Listening on port ${server.port}`)
```

## Custom Bun.serve options

Pass low-level `Bun.serve` options through `app.listen`:

```ts
await app.listen(3000, {
  maxRequestBodySize: 50 * 1024 * 1024,   // 50 MB
  lowMemoryMode: true,
  tls: {
    cert: Bun.file('./cert.pem'),
    key: Bun.file('./key.pem'),
  },
})
```
