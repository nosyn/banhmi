# Bun.password

`Bun.password` is Bun's built-in bcrypt/argon2 implementation — written in
Zig for maximum performance without native addon dependencies.

## When to use

- Hash user passwords before storing them in the database.
- Verify a plaintext password against a stored hash during login.

## Usage

```ts
import { Injectable } from 'banhmi'

@Injectable()
export class AuthService {
  async hashPassword(password: string): Promise<string> {
    return Bun.password.hash(password, {
      algorithm: 'argon2id',
      memoryCost: 4,    // MiB
      timeCost: 3,
    })
  }

  async verifyPassword(password: string, hash: string): Promise<boolean> {
    return Bun.password.verify(password, hash)
  }
}
```

## Algorithm options

| Algorithm | Notes |
|-----------|-------|
| `argon2id` | Recommended — memory-hard, resistant to GPU cracking |
| `argon2i` | Memory-hard, optimised for side-channel resistance |
| `argon2d` | Data-dependent — fastest, not recommended for passwords |
| `bcrypt` | Legacy support; fewer parameters |

## Performance

Bun's Zig implementation runs argon2id ~2–4x faster than the `argon2` Node.js
native addon, without requiring a build step.

## API reference

- `Bun.password.hash(password, options?)` — returns a PHC-format hash string (async).
- `Bun.password.hashSync(password, options?)` — synchronous version.
- `Bun.password.verify(password, hash)` — returns `boolean` (async).
- `Bun.password.verifySync(password, hash)` — synchronous version.
- Options: `algorithm` (`'argon2id'|'argon2i'|'argon2d'|'bcrypt'`), `memoryCost`, `timeCost`.
