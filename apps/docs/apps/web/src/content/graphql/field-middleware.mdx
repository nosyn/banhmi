# Field Middleware

Field middleware runs around individual field resolver calls, similar to how
HTTP interceptors wrap handler calls. Use them for field-level logging,
authentication, or transformation.

## Usage

```ts
import type { FieldMiddleware, MiddlewareContext, NextFn } from '@banhmi/graphql'

export const logMiddleware: FieldMiddleware = async (
  ctx: MiddlewareContext,
  next: NextFn,
) => {
  const start = Date.now()
  const result = await next()
  console.log(`[GQL] ${ctx.info.fieldName} → ${Date.now() - start}ms`)
  return result
}
```

Register globally in `GraphQLModule`:

```ts
GraphQLModule.forRoot({
  autoSchemaFile: true,
  fieldMiddleware: [logMiddleware],
})
```

Or per-field:

```ts
@Field({ middleware: [logMiddleware] })
name: string
```

## Guard-like field middleware

```ts
export const authMiddleware: FieldMiddleware = async (ctx, next) => {
  const user = ctx.context.user
  if (!user) throw new ForbiddenError('Not authenticated')
  return next()
}
```

## API reference

- `FieldMiddleware` — type: `(ctx: MiddlewareContext, next: NextFn) => Promise<unknown>`.
- `MiddlewareContext` — `{ source, args, context, info }`.
- `NextFn` — `() => Promise<unknown>` — calls the next middleware or resolver.
- `fieldMiddleware` option — global list applied to every resolved field.
- `@Field({ middleware: [...] })` — field-specific middleware list.
