# Extensions

GraphQL `extensions` are an arbitrary key-value map attached to types or
fields in the schema. They are useful for passing metadata to Apollo Studio,
tracing tools, or custom schema processors.

## Type extensions

```ts
import { Extensions, ObjectType } from '@banhmi/graphql'

@ObjectType()
@Extensions({ complexity: 1, role: 'admin' })
export class Cat {
  @Field()
  @Extensions({ cacheControl: { maxAge: 30 } })
  name: string
}
```

## Resolver extensions

```ts
import { Extensions, Query, Resolver } from '@banhmi/graphql'

@Resolver()
export class CatsResolver {
  @Query(() => [Cat])
  @Extensions({ complexity: 5 })
  cats() {
    return this.catsService.findAll()
  }
}
```

## Reading extensions at runtime

Extensions are accessible in plugins and middleware via `info.parentType.extensions`
and `info.fieldNodes[0].directives`.

## API reference

- `@Extensions(record)` — attach arbitrary metadata to a type, field, or resolver.
- Extensions are merged into the generated SDL `extensions` object.
- Access via GraphQL-js `GraphQLObjectType.extensions` at runtime.
