# Generating SDL

Banhmi can generate and persist the GraphQL SDL (Schema Definition Language)
file from your code-first type definitions. This is useful for tooling,
federation registry uploads, and API documentation.

## Auto-generate to file

```ts
GraphQLModule.forRoot({
  autoSchemaFile: 'schema.gql',  // write to disk at startup
})
```

A `schema.gql` file is created / updated each time the app starts.

## In-memory (no file)

```ts
GraphQLModule.forRoot({
  autoSchemaFile: true,  // keep schema in memory only
})
```

## Standalone script

Generate the schema without starting the HTTP server:

```ts
// scripts/generate-schema.ts
import { GraphQLSchemaFactory } from '@banhmi/graphql'
import { printSchema } from 'graphql'
import { writeFileSync } from 'fs'

const factory = new GraphQLSchemaFactory()
await factory.create([CatsResolver, /* ... */])
const schema = factory.schema
writeFileSync('schema.gql', printSchema(schema))
```

Run with:

```bash
bun run scripts/generate-schema.ts
```

## SDL-first (not recommended)

Banhmi's primary approach is code-first. If you need SDL-first, pass the
schema path via `typePaths`:

```ts
GraphQLModule.forRoot({
  typePaths: ['./src/**/*.graphql'],
})
```

## API reference

- `autoSchemaFile: true | 'path.gql'` — controls schema file generation.
- `sortSchema: true` — sort the generated SDL alphabetically.
- `GraphQLSchemaFactory` — build the schema programmatically without a running app.
