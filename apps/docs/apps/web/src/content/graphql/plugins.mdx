# Plugins

Banhmi's GraphQL engine accepts any Apollo Server plugin. Plugins hook into
the request lifecycle to add logging, tracing, error reporting, and more.

## When to use

- Send GraphQL operation telemetry to Sentry, Datadog, or Apollo Studio.
- Enforce query depth limits or complexity budgets at the request level.
- Log all operations in development for debugging.

## Built-in plugins

`@banhmi/graphql` ships an `IntrospectionPlugin` (disables introspection
in production) and a `LoggingPlugin` (logs every operation and its duration).

## Custom plugin

```ts
import type { ApolloServerPlugin, GraphQLRequestContext } from '@apollo/server'

export const myPlugin: ApolloServerPlugin = {
  async requestDidStart(ctx: GraphQLRequestContext<Record<string, unknown>>) {
    const start = Date.now()
    return {
      async willSendResponse() {
        console.log(`[GQL] ${ctx.request.operationName} → ${Date.now() - start}ms`)
      },
    }
  },
}
```

Register in `GraphQLModule`:

```ts
GraphQLModule.forRoot({
  autoSchemaFile: true,
  plugins: [myPlugin],
})
```

## API reference

- `plugins` option — array of `ApolloServerPlugin` instances.
- `ApolloServerPlugin` — interface from `@apollo/server`.
- Key lifecycle hooks: `requestDidStart`, `executionDidStart`, `willSendResponse`.
