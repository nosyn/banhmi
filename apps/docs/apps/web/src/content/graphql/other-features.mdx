# Other Features

This page covers miscellaneous `@banhmi/graphql` features not covered
by dedicated pages.

## Deprecation

```ts
@Field({ deprecationReason: 'Use `displayName` instead' })
name: string
```

## Default values

```ts
@Field(() => Int, { defaultValue: 0 })
score: number
```

## Nullable fields

```ts
@Field(() => String, { nullable: true })
bio?: string

// Nullable list
@Field(() => [String], { nullable: 'items' })
tags: (string | null)[]

// Nullable list and items
@Field(() => [String], { nullable: 'itemsAndList' })
metadata?: (string | null)[]
```

## Description

```ts
@ObjectType({ description: 'Represents a cat in the system' })
export class Cat {
  @Field({ description: "The cat's given name" })
  name: string
}
```

## Context typing

Type the GraphQL context for all resolvers in one place:

```ts
// graphql-context.ts
export interface GqlContext {
  user: User | null
  requestId: string
}

// In module config
GraphQLModule.forRoot<ApolloDriverConfig>({
  context: ({ req }) => ({
    user: req.user,
    requestId: req.headers.get('x-request-id'),
  }),
})
```
