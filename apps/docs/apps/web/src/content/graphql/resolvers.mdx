# Resolvers

Resolvers in `@banhmi/graphql` are TypeScript classes that map GraphQL
operations (queries, mutations, field resolutions) to provider methods.

## Query resolver

```ts
import { Args, Query, Resolver } from '@banhmi/graphql'
import { CatsService } from './cats.service'
import { Cat } from './models/cat.model'

@Resolver(() => Cat)
export class CatsResolver {
  static inject = [CatsService] as const
  constructor(private catsService: CatsService) {}

  @Query(() => [Cat])
  cats(): Cat[] {
    return this.catsService.findAll()
  }
}
```

## Field resolver

Resolve computed or related fields without embedding logic in the type class:

```ts
import { ResolveField, Parent } from '@banhmi/graphql'
import { Owner } from '../owners/models/owner.model'

@Resolver(() => Cat)
export class CatsResolver {
  @ResolveField(() => Owner, { nullable: true })
  owner(@Parent() cat: Cat) {
    return this.ownersService.findByCatId(cat.id)
  }
}
```

## Context

Access the GraphQL context (request, user, dataloaders) via `@Context`:

```ts
import { Context } from '@banhmi/graphql'

@Query(() => User)
me(@Context('user') user: User) {
  return user
}
```

## DataLoader pattern

```ts
@ResolveField(() => [Toy])
toys(@Parent() cat: Cat, @Context('loaders') loaders: Loaders) {
  return loaders.toys.load(cat.id)
}
```

## API reference

- `@Resolver(() => Type)` — binds the class to a GraphQL object type.
- `@Query(() => Type, opts?)` — declares a top-level query.
- `@Mutation(() => Type, opts?)` — declares a top-level mutation.
- `@ResolveField(() => Type, opts?)` — resolves a single field on the parent type.
- `@Parent()` — injects the parent object into a field resolver.
- `@Args(name, opts?)` / `@ArgsType()` — extract arguments.
- `@Context(key?)` — extract value(s) from the GraphQL context object.
