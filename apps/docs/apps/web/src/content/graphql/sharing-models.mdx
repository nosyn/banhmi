# Sharing Models

A class decorated with both `@ObjectType()` and `@InputType()` can serve as
both a GraphQL output type and an input type, reducing duplication.

## Shared class

```ts
import { Field, InputType, ObjectType } from '@banhmi/graphql'

@ObjectType()
@InputType('CatInput')
export class Cat {
  @Field(() => String, { nullable: true })
  id?: string

  @Field()
  name: string

  @Field(() => Number)
  age: number
}
```

The `@InputType('CatInput')` argument provides a distinct name for the input
type in the schema (required because GraphQL object and input types can't share names).

## Best practice

Prefer separate `@ObjectType()` and `@InputType()` classes for non-trivial
schemas. Shared models are convenient for simple cases but can lead to
schema fields (like `id`) accidentally being exposed as writeable inputs.
