# Execution Context

The `ExecutionContext` interface extends `ArgumentsHost` with handler-introspection
methods. Guards, interceptors, and exception filters all receive an
`ExecutionContext` so they can inspect the current request regardless of transport.

## When to use

- Inside guards/interceptors that must work across HTTP, WebSocket, and microservice contexts.
- To read `@SetMetadata` values attached to the current handler or controller.

## Usage — switching context

```ts
import type { ExecutionContext } from 'banhmi'

function getRequest(ctx: ExecutionContext): Request {
  const type = ctx.getType<'http' | 'ws' | 'rpc'>()
  if (type === 'http') {
    return ctx.switchToHttp().getRequest<Request>()
  }
  if (type === 'ws') {
    return ctx.switchToWs().getClient()
  }
  return ctx.switchToRpc().getData()
}
```

## Usage — reading metadata

```ts
import { Reflector } from 'banhmi'
import type { CanActivate, ExecutionContext } from 'banhmi'

@Injectable()
export class RolesGuard implements CanActivate {
  static inject = [Reflector] as const
  constructor(private reflector: Reflector) {}

  canActivate(ctx: ExecutionContext): boolean {
    const roles = this.reflector.getAllAndOverride<string[]>('roles', [
      ctx.getHandler(),
      ctx.getClass(),
    ])
    if (!roles) return true
    const user = ctx.switchToHttp().getRequest<Request & { user: User }>().user
    return roles.some((r) => user.roles.includes(r))
  }
}
```

## API reference

- `ctx.getType()` — returns `'http' | 'ws' | 'rpc'`.
- `ctx.getHandler()` — the route handler function.
- `ctx.getClass()` — the controller class.
- `ctx.switchToHttp()` — `{ getRequest, getResponse }`.
- `ctx.switchToWs()` — `{ getClient, getData }`.
- `ctx.switchToRpc()` — `{ getContext, getData }`.
- `Reflector.get(key, target)` / `getAllAndOverride(key, targets)` — read metadata.
