# Platform Agnosticism

Banhmi's core (`@banhmi/core`) is transport-agnostic. It defines the
`HttpAdapter` interface; `@banhmi/platform-bun` ships the default implementation
using raw `Bun.serve`. You can substitute your own adapter.

## Adapter interface

```ts
export interface HttpAdapter {
  listen(port: number): Promise<void>
  close(): Promise<void>
  getRequestMethod(request: unknown): string
  getRequestUrl(request: unknown): string
  reply(response: unknown, body: unknown, statusCode: number): void
  // ... other lifecycle hooks
}
```

## Default adapter — `BunAdapter`

`@banhmi/platform-bun` wires `Bun.serve` into the adapter interface and uses
the Radix-tree router for sub-millisecond route lookup.

## Custom adapter

```ts
import type { HttpAdapter } from 'banhmi'

class MyAdapter implements HttpAdapter {
  async listen(port: number) { /* ... */ }
  async close() { /* ... */ }
  // implement remaining methods
}

const app = await BanhmiFactory.create(AppModule, new MyAdapter())
await app.listen(3000)
```

## Why this matters

- Unit-testing: swap the adapter for an in-process stub without binding a port.
- Alternative runtimes: a Deno adapter or a Node.js adapter can be plugged in
  without touching application code.
- Edge workers: `@banhmi/edge` ships a fetch-based adapter for Cloudflare Workers
  and Bun edge deployments.

## API reference

- `HttpAdapter` — the interface all adapters must implement.
- `BunAdapter` — shipped by `@banhmi/platform-bun`; used by default.
- `BanhmiFactory.create(AppModule, adapter?)` — pass a custom adapter as the second argument.
