# Dynamic Modules

Dynamic modules return a `DynamicModule` object from a static factory method
(`forRoot`, `forFeature`, `register`, `forRootAsync`) so callers can pass
configuration at import time instead of hard-coding it.

## When to use

- Build reusable library modules where the host application provides options.
- Support both sync (`forRoot(options)`) and async (`forRootAsync({ useFactory })`)
  configuration.
- Expose a sub-set of providers to feature modules (`forFeature`).

## Sync pattern

```ts
import { DynamicModule, Module } from 'banhmi'
import { Token } from 'banhmi'

export const MAILER_OPTIONS_TOKEN = Token<MailerOptions>('MAILER_OPTIONS')

@Module({})
export class MailerModule {
  static forRoot(options: MailerOptions): DynamicModule {
    return {
      module: MailerModule,
      providers: [
        { provide: MAILER_OPTIONS_TOKEN, useValue: options },
        MailerService,
      ],
      exports: [MailerService],
    }
  }
}
```

```ts
@Module({
  imports: [MailerModule.forRoot({ host: 'smtp.example.com', port: 587 })],
})
export class AppModule {}
```

## Async pattern

```ts
@Module({})
export class MailerModule {
  static forRootAsync(options: {
    useFactory: (...args: unknown[]) => MailerOptions | Promise<MailerOptions>
    inject?: unknown[]
  }): DynamicModule {
    return {
      module: MailerModule,
      providers: [
        {
          provide: MAILER_OPTIONS_TOKEN,
          useFactory: options.useFactory,
          inject: options.inject ?? [],
        },
        MailerService,
      ],
      exports: [MailerService],
    }
  }
}
```

## API reference

- `DynamicModule` — `{ module, imports?, providers?, exports?, controllers?, global? }`.
- `forRoot(options)` — sync configuration.
- `forRootAsync({ useFactory, inject })` — async configuration.
- `forFeature(features)` — sub-module registration (e.g. per-repository).
- `global: true` — makes the dynamic module globally available.
