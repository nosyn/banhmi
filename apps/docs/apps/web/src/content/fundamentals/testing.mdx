# Testing

Banhmi tests run under the Bun test runner (`bun test`). The `@banhmi/testing`
package provides a `TestingModule` builder that wires the DI container without
starting an HTTP server.

## Setup

```bash
bun add -d @banhmi/testing
```

## Unit test

```ts
import { describe, expect, test } from 'bun:test'
import { TestingModule } from '@banhmi/testing'
import { CatsService } from './cats.service'

describe('CatsService', () => {
  let service: CatsService

  test('should be defined', async () => {
    const module = await TestingModule.createTestingModule({
      providers: [CatsService],
    }).compile()

    service = module.get(CatsService)
    expect(service).toBeDefined()
  })
})
```

## Mock providers

```ts
const mockCatsService = { findAll: () => [{ id: '1', name: 'Tom' }] }

const module = await TestingModule.createTestingModule({
  controllers: [CatsController],
  providers: [
    { provide: CatsService, useValue: mockCatsService },
  ],
}).compile()

const controller = module.get(CatsController)
expect(await controller.findAll()).toHaveLength(1)
```

## Integration test (HTTP)

```ts
import { BanhmiFactory } from 'banhmi'

test('GET /cats returns 200', async () => {
  const app = await BanhmiFactory.create(AppModule)
  const server = await app.listen(0)  // random port

  const res = await fetch(`http://localhost:${server.port}/cats`)
  expect(res.status).toBe(200)

  await app.close()
})
```

## API reference

- `TestingModule.createTestingModule(metadata)` — mirror of `@Module()` metadata.
- `.compile()` — builds the DI container; returns a `TestingModule` instance.
- `module.get(Token)` — retrieve a provider by class or token.
- `module.resolve(Token)` — create a new request-scoped or transient instance.
- `module.createNestApplication()` — wrap in a full application for e2e tests.
