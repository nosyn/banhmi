# Custom Providers

Beyond simple class providers, Banhmi supports value providers, factory providers,
class-alias providers, and existing-provider aliases — all through the same
`{ provide, use* }` object syntax.

## When to use

- Bind third-party values or instances that cannot be decorated with `@Injectable()`.
- Build providers whose construction requires async work (use `useFactory`).
- Alias one token to another (existing providers).
- Replace an implementation for testing without changing imports.

## Value provider

```ts
const mockCatsService = { findAll: () => [] }

@Module({
  providers: [
    { provide: CatsService, useValue: mockCatsService },
  ],
})
export class AppModule {}
```

## Class provider (alias)

```ts
@Module({
  providers: [
    { provide: CatsService, useClass: MockCatsService },
  ],
})
export class TestModule {}
```

## Factory provider

```ts
@Module({
  providers: [
    {
      provide: DATABASE_TOKEN,
      useFactory: async (config: ConfigService) => {
        const db = new Database(config.get('DB_URL'))
        await db.connect()
        return db
      },
      inject: [ConfigService],
    },
  ],
})
export class DatabaseModule {}
```

## Existing provider

```ts
@Module({
  providers: [
    LoggerService,
    { provide: 'ALIAS_LOGGER', useExisting: LoggerService },
  ],
})
export class AppModule {}
```

## Typed tokens

```ts
import { Token } from 'banhmi'

export const DB_TOKEN = Token<Database>('DB_TOKEN')

// Read in a constructor:
static inject = [DB_TOKEN] as const
constructor(private db: Database) {}
```

## API reference

- `useValue` — a static value bound to the token at module init time.
- `useClass` — a class instantiated by the DI container.
- `useFactory` — an (optionally async) factory function; `inject` lists its deps.
- `useExisting` — creates an alias pointing to an already-registered token.
- `Token<T>(name)` — create a typed, self-documenting injection token.
