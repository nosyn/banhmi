# Discovery Service

The `DiscoveryService` (from `@banhmi/core`) scans all registered providers and
controllers in the application for decorators applied with `Symbol.metadata`.
It powers features like `@EventPattern`, `@MessagePattern`, `@Cron`, and
`@SubscribeMessage` by finding handlers at startup time.

## When to use

- Building framework extensions or plugins that scan for custom decorators.
- Implementing dynamic handler registration (event emitters, schedulers, queues).
- Introspecting the provider graph for documentation or tooling purposes.

## Setup

`DiscoveryService` is exported from `@banhmi/core` and available without additional
module imports if you import `CoreModule` (included by default).

## Usage

```ts
import { DiscoveryService, Injectable, OnModuleInit } from 'banhmi'

const HANDLER_SYMBOL = Symbol('handler')

function Handle(event: string) {
  return (_target: unknown, context: ClassMethodDecoratorContext) => {
    context.metadata[HANDLER_SYMBOL] = event
  }
}

@Injectable()
export class EventBus implements OnModuleInit {
  static inject = [DiscoveryService] as const
  constructor(private discovery: DiscoveryService) {}

  onModuleInit() {
    const handlers = this.discovery.getProvidersWithMetaAtKey(HANDLER_SYMBOL)
    for (const { instance, meta } of handlers) {
      console.log(`Registered handler for event: ${meta}`)
    }
  }
}
```

## API reference

- `DiscoveryService.getProvidersWithMetaAtKey(symbol)` — returns `{ instance, meta }[]`.
- `DiscoveryService.getControllersWithMetaAtKey(symbol)` — same, for controllers.
- `DiscoveryService.getHandlersWithMetaAtKey(symbol)` — finds method-level metadata.
- `DiscoveryService.getProviders()` — all registered provider wrappers.
- `DiscoveryService.getControllers()` — all registered controller wrappers.
