# Circular Dependency

A circular dependency occurs when module A imports module B and module B imports
module A, or when provider A depends on provider B and vice versa. Banhmi detects
these at startup and throws a descriptive error.

## When this happens

- Two feature modules mutually import each other.
- Two services each list the other in their `static inject`.

## Resolution — `forwardRef`

Use `forwardRef()` to break the cycle by deferring resolution:

```ts
// Module level
@Module({
  imports: [forwardRef(() => CatsModule)],
})
export class DogsModule {}
```

```ts
// Provider level
@Injectable()
export class CatsService {
  static inject = [forwardRef(() => DogsService)] as const
  constructor(private dogsService: DogsService) {}
}
```

## Better approach — refactoring

Forward references are a code smell. Prefer extracting shared logic into a
third `SharedModule` that both modules import:

```
CatsModule ──imports──► SharedModule
DogsModule ──imports──► SharedModule
```

## API reference

- `forwardRef(() => Token)` — defers token resolution until after both sides initialise.
