# Streaming Files

Streaming file responses bypass in-memory buffering and are ideal for large
files, downloads, and video/audio content. Bun's native file API integrates
directly with `Response`.

## When to use

- Serve large file downloads without holding them in memory.
- Stream logs, CSV exports, or generated content to the client.
- Proxy a file from S3 or another upstream source.

## Serve a local file

```ts
import { Controller, Get, Param } from 'banhmi'

@Controller('files')
export class FilesController {
  @Get(':filename')
  download(@Param('filename') filename: string) {
    const file = Bun.file(`./uploads/${filename}`)
    // Bun.file() is a BunFile — Response accepts it directly
    return new Response(file, {
      headers: {
        'Content-Disposition': `attachment; filename="${filename}"`,
        'Content-Type': file.type,
      },
    })
  }
}
```

## Stream a ReadableStream

```ts
@Get('export')
exportCsv() {
  const stream = new ReadableStream({
    start(controller) {
      controller.enqueue('id,name\n')
      controller.enqueue('1,Tom\n')
      controller.enqueue('2,Jerry\n')
      controller.close()
    },
  })

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/csv',
      'Content-Disposition': 'attachment; filename="export.csv"',
    },
  })
}
```

## Proxy from S3

```ts
@Get('report/:key')
async proxyFromS3(@Param('key') key: string) {
  const s3File = Bun.s3.file(key)           // Bun 1.1+ S3 API
  return new Response(s3File)
}
```

## API reference

- `Bun.file(path)` — returns a `BunFile` usable as a `Response` body.
- `new Response(stream | bunFile, init?)` — passes a stream or file directly.
- `Content-Disposition: attachment` — triggers browser download.
- `BunFile.type` — inferred MIME type from the file extension.
