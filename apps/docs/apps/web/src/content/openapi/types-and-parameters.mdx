# Types and Parameters

This page covers how `@banhmi/openapi` maps TypeScript types to OpenAPI 3.1.0
schemas, and how to document path and query parameters.

## Type mapping

`@ApiProperty` accepts a `type` string or a class reference:

| TypeScript / value | `type` option | OpenAPI schema |
|--------------------|---------------|---------------|
| `string`           | `'string'`    | `{ type: 'string' }` |
| `number` (default) | `'number'`    | `{ type: 'number', format: 'float' }` |
| `number` (integer) | `'number'` + `format: 'int'` | `{ type: 'integer' }` |
| `boolean`          | `'boolean'`   | `{ type: 'boolean' }` |
| `string[]`         | `['string']`  | `{ type: 'array', items: { type: 'string' } }` |
| Class              | `MyClass`     | `{ $ref: '#/components/schemas/MyClass' }` |

## Path parameters

```ts
@ApiParam({ name: 'id', type: 'string', description: 'Resource ID', required: true })
@Get('/:id')
findOne() {}
```

## Query parameters

```ts
@ApiQuery({ name: 'limit', type: 'number', required: false, description: 'Max results' })
@ApiQuery({ name: 'offset', type: 'number', required: false })
@Get()
findAll() {}
```

## Request body

```ts
@ApiBody({ type: CreateCatDto, description: 'Cat data', required: true })
@Post()
create() {}
```
