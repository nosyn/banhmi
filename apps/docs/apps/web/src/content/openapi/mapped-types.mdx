# Mapped Types

Mapped types let you derive DTO variants (e.g. `PartialType`, `PickType`,
`OmitType`) that carry over `@ApiProperty` metadata. This feature is **planned**
for a future wave; the current Wave 5 ships the decorator primitives on which
mapped types will be built.

## Planned API

```ts
// Future wave — not yet available
import { PartialType, PickType, OmitType } from '@banhmi/openapi'

class UpdateCatDto extends PartialType(CreateCatDto) {}
class CatSummaryDto extends PickType(CreateCatDto, ['name'] as const) {}
```

## Current workaround

Manually re-declare the DTO with the desired properties and `@ApiProperty`
decorators:

```ts
class UpdateCatDto {
  @ApiProperty({ type: 'string', required: false })
  name?: string

  @ApiProperty({ type: 'number', required: false })
  age?: number
}
```
