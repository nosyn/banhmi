# CLI Plugin

The `@banhmi/openapi/cli` plugin automatically injects `@ApiProperty()` decorators
on class properties that have explicit TypeScript type annotations but no existing
`@ApiProperty`, `@ApiHideProperty`, or `@Exclude` decorator.

This mirrors the behaviour of the NestJS CLI plugin — write plain DTOs and get
fully documented OpenAPI schemas without manual decoration.

## When to use

Use the plugin when you have many DTO classes and don't want to annotate every
property by hand. The plugin is conservative: it only annotates properties with
explicit `: Type` annotations and skips everything else.

## Setup

### Option 1 — `bunfig.toml` preload (recommended)

```toml
# bunfig.toml
preload = ["@banhmi/openapi/cli"]
```

### Option 2 — Programmatic

```ts
import '@banhmi/openapi/cli'
```

## How it works

The plugin registers a Bun file-loader hook that scans each TypeScript source file
for annotatable properties and inserts `@ApiProperty()` before them at load time.
The original file is never modified on disk.

**Annotated:**
```ts
class Cat {
  name: string         // → @ApiProperty() name: string
  age?: number         // → @ApiProperty() age?: number
}
```

**Skipped (already decorated):**
```ts
class Cat {
  @ApiProperty({ example: 'Whiskers' })
  name: string         // unchanged

  @ApiHideProperty()
  secret: string       // unchanged

  @Exclude()
  internal: string     // unchanged
}
```

**Skipped (access modifiers):**
```ts
class Cat {
  private id: number   // skipped — private
  static count: number // skipped — static
  readonly tag: string // skipped — readonly
}
```

## `transformSource` function

For unit-testing or custom tooling, the underlying transform is exported as a
pure function:

```ts
import { transformSource } from '@banhmi/openapi/cli'

const output = transformSource(`class A { name: string }`)
// → 'class A {\n  @ApiProperty()\n  name: string\n}'
```
