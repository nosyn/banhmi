# Operations

An *operation* in OpenAPI corresponds to a single route handler. Use `@ApiOperation`
and `@ApiResponse` to document what each handler does.

## Describing an operation

```ts
@ApiOperation({
  summary: 'Get a cat by ID',
  description: 'Returns a single cat. Throws 404 if not found.',
  deprecated: false,
})
@Get('/:id')
findOne() {}
```

## Documenting responses

Stack multiple `@ApiResponse` decorators for each possible HTTP status:

```ts
@ApiResponse({ status: 200, description: 'Cat found', type: Cat })
@ApiResponse({ status: 404, description: 'Cat not found' })
@ApiResponse({ status: 401, description: 'Unauthorised' })
@Get('/:id')
findOne() {}
```

## Excluding an endpoint

Use `@ApiExcludeEndpoint()` to hide an internal handler from the generated spec:

```ts
@ApiExcludeEndpoint()
@Get('/healthz')
health() {
  return { ok: true }
}
```
