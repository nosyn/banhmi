# Examples Catalog

Each example is a standalone Banhmi application that you can clone and run
with `bun run dev`. Browse the source on GitHub or clone the repo and run
from the `examples/` directory.

## Cluster apps

| Example | Description | Key features |
|---------|-------------|-------------|
| [`cats-api`](https://github.com/banhmi/banhmi/tree/main/examples/cats-api) | Classic CRUD API | Controllers, services, pipes, validation |
| [`cats-mongo`](https://github.com/banhmi/banhmi/tree/main/examples/cats-mongo) | MongoDB-backed CRUD | `@banhmi/mongo`, Mongoose integration |
| [`drizzle-api`](https://github.com/banhmi/banhmi/tree/main/examples/drizzle-api) | Postgres with Drizzle ORM | `@banhmi/drizzle`, type-safe SQL |

## Feature examples (`examples/features/`)

Each feature example is a minimal single-file (or small multi-file) app
demonstrating one specific capability:

- `compression` — gzip/zstd response compression
- `cookies` — cookie read/write
- `cors` — CORS configuration
- `csrf` — CSRF token protection
- `crypto` — HMAC and AES helpers
- `file-upload` — multipart form upload
- `helmet` — security headers
- `rate-limiting` — request throttling
- `serialization` — `ClassSerializerInterceptor`
- `serve-static` — static file serving
- `session` — server-side sessions
- `sse` — Server-Sent Events
- `validation-native` — built-in `ValidationPipe`
- `validation-zod` — Zod-based validation

## Running an example

```bash
cd examples/cats-api
bun install   # only needed first time
bun run dev
```
