# Examples Catalog

Each example is a standalone Banhmi application that you can clone and run
with `bun run dev`. Browse the source on GitHub or clone the repo and run
from the `examples/` directory.

## Recommended starting point

[`kitchen-sink`](https://github.com/banhmi/banhmi/tree/main/examples/kitchen-sink)
is the recommended end-to-end demo. It wires 20+ Banhmi packages into a single
runnable task-tracker API ŌĆö the fastest way to see what the framework can do:

```bash
cd examples/kitchen-sink && bun run dev
# API docs:  http://localhost:3000/api/docs
# Health:    http://localhost:3000/api/health
# Devtools:  http://localhost:3000/__banhmi/devtools
```

Features demonstrated: HTTP versioning, class-validator DTOs, JWT auth,
throttling, SQLite, EventEmitter, SSE, WebSocket, queues, scheduling,
caching, serialization, multipart uploads, static file serving, compression,
signed cookies, security headers, health checks, OpenAPI/Scalar, and devtools.

## Feature examples (`examples/features/`)

Each feature example is a minimal single-file (or small multi-file) app
demonstrating one specific capability:

- `compression` ŌĆö gzip/zstd response compression
- `cookies` ŌĆö cookie read/write
- `cors` ŌĆö CORS configuration
- `csrf` ŌĆö CSRF token protection
- `crypto` ŌĆö HMAC and AES helpers
- `file-upload` ŌĆö multipart form upload
- `helmet` ŌĆö security headers
- `rate-limiting` ŌĆö request throttling
- `serialization` ŌĆö `ClassSerializerInterceptor`
- `serve-static` ŌĆö static file serving
- `session` ŌĆö server-side sessions
- `sse` ŌĆö Server-Sent Events
- `validation-native` ŌĆö built-in `ValidationPipe`
- `validation-zod` ŌĆö Zod-based validation
