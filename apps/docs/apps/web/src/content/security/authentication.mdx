# Authentication

Banhmi ships two first-class authentication packages. Choose the one that fits
your application's needs.

## Choose your auth philosophy

### `@banhmi/auth` — Passport-style strategies

Compose individual strategies (Local, JWT Bearer, Google OAuth, GitHub OAuth)
onto specific routes using `@UseAuth`. You control the session format, token
lifetime, and principal shape.

```bash
bun add @banhmi/auth
```

**When to pick `@banhmi/auth`:**
- You already have a JWT issuer or a custom credential store.
- You want to mix multiple auth mechanisms on different routes.
- You need a lightweight, zero-session setup (stateless JWT).

### `@banhmi/better-auth` — end-to-end sessions

[better-auth](https://better-auth.com) owns sessions, providers, magic links,
RBAC, and 30+ OAuth providers. `@banhmi/better-auth` wires it into the Banhmi
DI container with `BetterAuthModule.forRoot()` and a `BetterAuthGuard` you
apply with `@UseGuards`.

```bash
bun add @banhmi/better-auth better-auth
```

**When to pick `@banhmi/better-auth`:**
- You want sessions, OAuth, magic links, and RBAC without building them yourself.
- You're migrating from an app that already uses better-auth.
- You prefer configuration-over-code for auth providers.

> **Which should I pick?**
> - Need only JWT + a login form? → `@banhmi/auth`.
> - Need Google/GitHub login, session management, and org roles out of the box? → `@banhmi/better-auth`.
> - Need both (e.g. API JWT + web session)? Use both — they don't conflict.

### Strategies

#### `LocalStrategy`

Reads `username` and `password` from a JSON body or
`application/x-www-form-urlencoded` form. Calls your `validate` callback to
authenticate the credentials.

```ts
import { LocalStrategy } from '@banhmi/auth'
import { verifyPassword } from '@banhmi/crypto'

const local = new LocalStrategy({
  validate: async ({ username, password }) => {
    const user = await db.findByUsername(username)
    if (!user) return null
    const ok = await verifyPassword(password, user.passwordHash)
    return ok ? user : null
  },
})
```

Custom field names:

```ts
new LocalStrategy({
  usernameField: 'email',
  passwordField: 'pwd',
  validate: async ({ username, password }) => { /* ... */ },
})
```

#### `JwtStrategy`

Verifies an HS256 JWT from the `Authorization: Bearer <token>` header using
`@banhmi/jwt` under the hood. Returns the decoded payload on success.

```ts
import { JwtStrategy } from '@banhmi/auth'

const jwt = new JwtStrategy({ secret: Bun.env.JWT_SECRET ?? '' })
```

Custom extractor (e.g., query parameter):

```ts
new JwtStrategy({
  secret: '...',
  extract: (req) => new URL(req.url).searchParams.get('token'),
})
```

#### `GoogleStrategy` / `GitHubStrategy`

OAuth 2.0 authorization-code flow. Redirects the user to the provider, then
exchanges the code on the callback route.

```ts
import { GoogleStrategy } from '@banhmi/auth'

const google = new GoogleStrategy({
  clientId: Bun.env.GOOGLE_CLIENT_ID ?? '',
  clientSecret: Bun.env.GOOGLE_CLIENT_SECRET ?? '',
  redirectUri: 'https://example.com/auth/google/callback',
  validate: async (profile) => db.upsertUser(profile),
})

// Initiate login
const url = await google.getAuthorizationUrl()
return Response.redirect(url)

// On callback route — @UseAuth('google') reads code+state from query string
```

### Applying authentication with `@UseAuth`

```ts
import { Controller, Get, Post } from 'banhmi'
import { UseAuth, getAuthUser } from '@banhmi/auth'

@Controller()
class AppController {
  @Post('/login')
  @UseAuth('local', [local])
  login(ctx: RouteCtx) {
    return { user: getAuthUser(ctx) }
  }

  @Get('/me')
  @UseAuth('jwt', [jwt])
  me(ctx: RouteCtx) {
    return { user: getAuthUser(ctx) }
  }
}
```

On success, the authenticated principal is stored on `ctx.state` and
accessible via `getAuthUser(ctx)`. On failure, the decorator returns
`401 Unauthorized` before the handler runs.

### `AuthModule` — DI integration

For DI-managed applications, register strategies via `AuthModule.register`:

```ts
import { AuthModule } from '@banhmi/auth'

@Module({
  imports: [
    AuthModule.register({ strategies: [local, jwt] }),
  ],
})
class AppModule {}
```

## `@banhmi/better-auth` — end-to-end sessions

### Setup

```ts
import { BetterAuthModule, BetterAuthGuard, getBetterAuthSession } from '@banhmi/better-auth'
```

Mount the better-auth HTTP handler first (in `main.ts`):

```ts
import { BanhmiFactory } from 'banhmi'
import { betterAuth } from 'better-auth'
import { Database } from 'bun:sqlite'

const auth = betterAuth({
  database: new Database('./auth.sqlite'),
  emailAndPassword: { enabled: true },
})

const app = await BanhmiFactory.create(AppModule)
app.use(async (req, next) => {
  if (new URL(req.url).pathname.startsWith('/api/auth')) {
    return auth.handler(req)
  }
  return next()
})
await app.listen(3001)
```

Register the plugin in your `AppModule`:

```ts
@Module({
  imports: [
    BetterAuthModule.forRoot({
      url: Bun.env.BETTER_AUTH_URL ?? 'http://localhost:3001',
    }),
  ],
})
class AppModule {}
```

### Protecting routes

```ts
import { BetterAuthGuard, getBetterAuthSession } from '@banhmi/better-auth'
import { Controller, Get, UseGuards } from 'banhmi'

@Controller('/users')
class UsersController {
  @Get('/me')
  @UseGuards(BetterAuthGuard)
  me(ctx: RouteCtx) {
    const session = getBetterAuthSession(ctx)
    return { user: session?.user }
  }

  @Get('/ping')
  ping() {
    return { ok: true }
  }
}
```

### Optional auth

On public routes that want to read the session when present:

```ts
@Get('/public')
public(ctx: RouteCtx) {
  const session = getBetterAuthSession(ctx) // null if no session
  return { loggedIn: session !== null }
}
```

## API reference — `@banhmi/auth`

### `UseAuth(strategyName, strategies)`

Method decorator. Runs the named strategy before the handler. On success, sets
`ctx.state['banhmi:auth:user']`. On failure, returns `401`.

### `getAuthUser<TUser>(ctx)`

Helper function. Returns the principal stored by `@UseAuth`, cast to `TUser`,
or `null` if not set.

### `Strategy<TUser>`

Abstract base class. Implement `name: string` and
`authenticate(req, ctx): Promise<TUser | null>` to create a custom strategy.

---

## API reference — `@banhmi/better-auth`

### `BetterAuthModule.forRoot(options)`

Registers the plugin. Provides `BETTER_AUTH_CLIENT`, `BETTER_AUTH_OPTIONS`, and
configures `BetterAuthGuard` in the DI container.

### `BetterAuthGuard`

A `Guard` that reads the session cookie, calls the better-auth `get-session`
endpoint, and throws `401 Unauthorized` if no valid session is found. Stores
the session on `ctx.state['banhmi:better-auth:session']` on success.

### `getBetterAuthSession(ctx)`

Helper that reads the session from `ctx.state`. Returns `null` if the guard
has not run or the session is absent.
