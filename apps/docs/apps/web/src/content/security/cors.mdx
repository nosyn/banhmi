# CORS

`CorsModule` handles Cross-Origin Resource Sharing (CORS). It responds to
preflight `OPTIONS` requests and sets the appropriate `Access-Control-*`
headers on all responses.

## When to use

When your API is served from a different origin than your front-end, or when
you need to control which domains are allowed to make requests.

## Setup

```bash
bun add @banhmi/security
```

## Configuration

```ts
CorsModule.forRoot({
  origin: 'https://app.example.com',   // string | string[] | RegExp | fn
  credentials: true,
  methods: ['GET', 'POST'],            // defaults to all standard methods
  allowedHeaders: ['authorization'],   // mirrors request headers when unset
  exposedHeaders: ['x-custom'],
  maxAge: 600,                         // preflight cache, seconds
})
```

## API reference

### `CorsModule.forRoot(opts?)`

`opts` is a `CorsOptions` object. When `origin` is omitted all origins are
allowed (`'*'`).
