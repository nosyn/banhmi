# Workspaces

Banhmi's CLI supports Bun workspaces out of the box. A workspace contains
multiple apps and libraries that share a single `bun.lock`.

## When to use

- Large teams building several services that share common utilities.
- Monorepo layouts where multiple Banhmi applications deploy independently.

## Setup

Root `package.json`:

```json
{
  "workspaces": ["apps/*", "packages/*"]
}
```

Then install all workspace deps from the root:

```bash
bun install
```

## Running workspace scripts

```bash
# Run in a specific workspace
bun run --filter apps/api dev

# Run in all workspaces that have the script
bun run --filter '*' test
```

## Cross-workspace imports

Workspace packages reference each other via `workspace:*`:

```json
{
  "dependencies": {
    "@myorg/shared": "workspace:*"
  }
}
```

## Banhmi workspace conventions

| Directory | Purpose |
|-----------|---------|
| `apps/` | Deployable Banhmi applications |
| `packages/` | Shared library packages |
| `tools/` | Build scripts, generators |

## bunfig.toml per workspace

Each package that uses decorators needs its own `bunfig.toml` preloading
the `Symbol.metadata` polyfill:

```toml
preload = ["../../node_modules/banhmi/polyfill-symbol-metadata"]
```
