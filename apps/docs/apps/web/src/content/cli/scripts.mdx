# Scripts

Scaffolded projects include a standard set of `package.json` scripts that
map directly to Bun and `banhmi` CLI commands.

## Standard scripts

```json
{
  "scripts": {
    "dev":       "banhmi start --watch",
    "start":     "banhmi start",
    "build":     "banhmi build",
    "lint":      "biome check --write .",
    "format":    "biome format --write .",
    "test":      "bun test",
    "test:cov":  "bun test --coverage",
    "type-check":"tsc --noEmit"
  }
}
```

## Quality scripts (workspace-level)

Projects generated from the Banhmi monorepo template include additional
quality gate scripts under `scripts/quality/`:

| Script | Purpose |
|--------|---------|
| `bun run quality:no-anys` | Fail if any `any` type remains |
| `bun run quality:no-bangs` | Fail if any `!` non-null assertion remains |
| `bun run quality:no-reflect` | Fail if `reflect-metadata` is imported |
| `bun run quality:tsdoc` | Check TSDoc coverage on public API |

Run all at once:

```bash
bun run lint && bun test && bun run type-check
```

## Custom scripts

Add custom scripts inline in `package.json`. For monorepo use, prefix
scripts with the workspace name:

```bash
bun run --filter @myorg/api dev
```
