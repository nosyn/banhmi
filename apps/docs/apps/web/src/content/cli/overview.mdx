# Overview

The `banhmi` CLI (`@banhmi/cli`) scaffolds projects and generates boilerplate
so you can focus on application logic rather than file setup.

## Installation

The CLI is bundled with the `banhmi` package. You can also install it globally:

```bash
bun add -g @banhmi/cli
```

## Commands

| Command | Description |
|---------|-------------|
| `banhmi new <name>` | Scaffold a new Banhmi application |
| `banhmi generate controller <name>` | Generate a controller |
| `banhmi generate service <name>` | Generate a service |
| `banhmi generate module <name>` | Generate a module |
| `banhmi generate resource <name>` | Generate a full CRUD resource (controller + service + module + DTOs) |
| `banhmi generate guard <name>` | Generate a guard |
| `banhmi generate interceptor <name>` | Generate an interceptor |
| `banhmi generate pipe <name>` | Generate a pipe |
| `banhmi generate filter <name>` | Generate an exception filter |
| `banhmi build` | Build the project with `bun build` |
| `banhmi start [--watch]` | Start the application |
| `banhmi info` | Display environment and package versions |

## Quick start

```bash
bun create banhmi my-app
cd my-app
bun run dev
```

## Alias

All commands accept the shorthand `bmi` alias:

```bash
bmi g controller users
bmi g service users
bmi g module users
```
