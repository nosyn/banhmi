# Libraries

Use `banhmi generate library` to scaffold a re-usable library package inside
a Banhmi workspace. Libraries are plain TypeScript packages without their own
`main.ts`; they export modules, providers, and utilities consumed by apps.

## When to use

- Extract shared business logic into a versioned, independently testable package.
- Build a plugin package (`@myorg/auth`, `@myorg/notifications`) that multiple
  apps can import.

## Generate a library

```bash
# Inside a Banhmi workspace
banhmi g library auth --prefix @myorg
```

Generated structure:

```
packages/
  auth/
    src/
      auth.module.ts
      auth.service.ts
      index.ts
    package.json    # name: "@myorg/auth", type: "module"
    tsconfig.json
    bunfig.toml
```

## Consume from an app

```ts
import { AuthModule } from '@myorg/auth'

@Module({
  imports: [AuthModule.forRoot({ secret: process.env.JWT_SECRET })],
})
export class AppModule {}
```

## Publishing

```bash
cd packages/auth
bun run build
bun publish
```

Libraries should set `"exports"` in `package.json` for proper ESM resolution:

```json
{
  "exports": {
    ".": "./src/index.ts"
  }
}
```
