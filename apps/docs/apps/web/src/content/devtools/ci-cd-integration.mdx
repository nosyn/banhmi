# CI/CD Integration

`@banhmi/devtools` is a **development-time tool** and must not be enabled in
production CI pipelines or deployed environments.

## Disabling in production

The middleware automatically disables itself when `NODE_ENV=production`:

```ts
DevtoolsModule.forRoot()
// Equivalent to: enabled: Bun.env.NODE_ENV !== 'production'
```

To be explicit, pass `enabled: false`:

```ts
DevtoolsModule.forRoot({ enabled: false })
```

## CI pipeline guidance

In CI you typically want `NODE_ENV=test` or `NODE_ENV=ci`. In that case
devtools will still be *enabled* (because it is not `'production'`). This is
intentional — integration tests can assert against the `/graph.json` endpoint
to verify the DI graph shape is correct as part of your test suite.

If you do not want devtools endpoints reachable during CI tests, set
`enabled: false` in the test app module or set `NODE_ENV=production` in the
CI environment.

```bash
# Disable devtools in CI:
NODE_ENV=production bun test
```

## What never to do

- Do **not** deploy with `enabled: true` in a public-facing environment. The
  endpoint exposes your full DI graph and recent request routes, which is
  sensitive information.
- Do **not** remove the `DevtoolsModule.forRoot()` import entirely from your
  production module tree if you rely on exported tokens — instead set
  `enabled: false`.
