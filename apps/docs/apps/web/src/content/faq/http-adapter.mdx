# HTTP Adapter

**Which HTTP adapter does Banhmi use?**

`@banhmi/platform-bun` ships `BunAdapter`, which wraps raw `Bun.serve`.
There are no Express, Fastify, or other framework dependencies.

**Can I use a different HTTP adapter?**

Yes — implement the `HttpAdapter` interface from `@banhmi/core` and pass an
instance directly:

```ts
import type { HttpAdapter } from '@banhmi/core'
import { BanhmiApplication, Container, ModuleGraph } from '@banhmi/core'

class MyAdapter implements HttpAdapter {
  registerController(instance, klass) { /* … */ }
  async listen(port) { /* … */ }
  async close() { /* … */ }
  use(mw) { /* … */ }
}

const graph = new ModuleGraph()
const moduleTree = graph.buildTree(AppModule)
const container = new Container()
// ... register providers ...
const adapter = new MyAdapter()
const app = new BanhmiApplication(container, moduleTree, adapter)
await app.listen(3000)
```

**Can I access the underlying `Bun.Server` instance?**

Yes — inject the adapter via `HTTP_ADAPTER_TOKEN` in an
`OnApplicationBootstrap` provider:

```ts
import { Injectable } from '@banhmi/common'
import { OnApplicationBootstrap } from '@banhmi/common'
import { HTTP_ADAPTER_TOKEN } from '@banhmi/platform-bun'
import type { BunAdapter } from '@banhmi/platform-bun'

@Injectable()
class SetupService implements OnApplicationBootstrap {
  static inject = [HTTP_ADAPTER_TOKEN] as const

  constructor(private adapter: BunAdapter) {}

  onApplicationBootstrap() {
    // adapter.dispatchRequest is available here
  }
}
```
