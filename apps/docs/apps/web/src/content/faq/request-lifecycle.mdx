# Request Lifecycle

**What is the order of execution for a request?**

```
pre-request middleware → module middleware → guards → interceptors (before)
→ pipes → handler → interceptors (after) → serialise → exception filters
```

See [Request Lifecycle](/deployment/request-lifecycle) for a full diagram and
stage descriptions.

**Where should I put authentication logic?**

In a **Guard**.  Guards run after all middleware but before interceptors and
the handler.  They have access to the full `ExecutionContext` including the
handler class and method metadata.

**Where should I put logging / timing?**

In an **Interceptor**.  The before-phase runs before the handler; the
after-phase (after `next.handle()`) runs after.

**Where should I put error formatting?**

In an **Exception Filter**.  Filters are the last stage; they convert any
thrown exception into a `Response`.

**Does middleware run before guards?**

Yes.  The order is: middleware → guards → interceptors → handler.  Middleware
can short-circuit the entire pipeline by returning a `Response` without calling
`next()`.
