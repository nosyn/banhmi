# Global Path Prefix

**How do I add a prefix like `/api/v1` to all routes?**

There is no `app.setGlobalPrefix()` method yet.  The current approaches are:

**Option 1 — Prefix controllers individually**

```ts
@Controller('/api/v1/cats')
class CatsController { /* … */ }
```

**Option 2 — Nest controllers under a module with a shared prefix**

Use `@Controller` on every controller that belongs to the same prefix group.

**Option 3 — Edge adapter `globalPrefix` option**

When deploying to an edge runtime via `@banhmi/edge`, you can strip an
incoming prefix before routing:

```ts
const handler = await createEdgeHandler(AppModule, { globalPrefix: 'api' })
```

Requests arriving at `/api/cats` are routed to the `/cats` controller.

**Is `app.setGlobalPrefix()` planned?**

Yes — it is on the roadmap.  Track progress in the framework's issue tracker.
