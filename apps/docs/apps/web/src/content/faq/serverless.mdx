# Serverless

**Can I run Banhmi on AWS Lambda or Vercel?**

Yes. Install `@banhmi/serverless`:

```bash
bun add @banhmi/serverless
```

Then create a handler:

```ts
// src/index.ts
import { createLambdaHandler } from '@banhmi/serverless'
import { AppModule } from './app.module'

export const handler = await createLambdaHandler(AppModule)
```

The handler works with API Gateway v1 (REST API) and v2 (HTTP API) event
formats — the adapter detects the format automatically.

**What about Cloudflare Workers / Vercel Edge?**

Use `@banhmi/edge` instead:

```ts
import { createEdgeHandler } from '@banhmi/edge'
import { AppModule } from './app.module'

export default { fetch: await createEdgeHandler(AppModule) }
```

**Does cold-start time matter?**

Both adapters bootstrap the full Banhmi lifecycle once (module init, DI,
controller registration) at handler creation time — not per invocation.
Warm Lambda invocations share the same bootstrapped instance, keeping
per-request overhead minimal.
